# Tapestry File Format — Canonical Description

> **Status:** Reverse-engineered from the public repo (`tapestry-project` at
> `ce06ade`, v6 schemas) and a production v7 export downloaded from
> `tapestries.media`. The public repo lags behind production.

---

## 1. Container

A Tapestry is a **ZIP file** (MIME `application/zip`). It contains:

```
whatever.tapestry.zip
├── root.json                    ← required: the full tapestry manifest
├── items/<uuid> (<name>).ext    ← media assets (images, audio, video, PDFs)
├── items/<uuid>-thumbnail ...   ← auto-generated item thumbnails
└── items/<uuid>-custom-...      ← user-uploaded custom thumbnails
```

Binary files are referenced from `root.json` with the `file:/` URI scheme followed by the relative path within the zip (e.g. `"file:/items/abc123 (name).jpg"`).

---

## 2. `root.json` — Root Object

| Field | Type | Required | Description |
|---|---|---|---|
| `version` | `6` or `7` (literal integer) | **yes** | Format version. v6 is in the public repo; v7 is deployed on `tapestries.media`. |
| `id` | `string` | **yes** | UUID of the tapestry |
| `title` | `string` (non-empty) | **yes** | |
| `description` | `string` or `null` | no | |
| `createdAt` | ISO 8601 date string | **yes** | e.g. `"2026-06-22T18:16:56"` |
| `updatedAt` | ISO 8601 date string | **yes** | Required in v6+. Same format as `createdAt`. |
| `items` | `Item[]` | **yes** | Array of item objects (can be empty) |
| `rels` | `Rel[]` | **yes** | Array of rel objects (can be empty) |
| `groups` | `Group[]` | **yes** | Array of group objects (can be empty) |
| `theme` | `"light"` or `"dark"` | **yes** | |
| `background` | hex color string | **yes** | `#RGB`, `#RGBA`, `#RRGGBB`, or `#RRGGBBAA` |
| `startView` | `{position: {x, y}, size: {width, height}}` or `null` | no | Initial viewport. v2+. |
| `thumbnail` | `string` or `null` | no | URL (usually `file:/`) of a tapestry-level thumbnail image |
| `parentId` | `string` or `null` | no | **v7 only.** ID of a parent/derived tapestry (forks?) |
| `presentation` | `PresentationStep[]` or `null` | no | **v7 only.** Ordered presentation/guided-tour steps. |

### ⚠️ Unknowns (v7)

- **`parentId`**: Appears in v7 exports. Likely relates to the fork/remix feature. Semantics not documented in the public repo.
- **`presentation`**: Array of `{id, prevStepId, type: "item"|"group", itemId?|groupId?}`. Present in v7 exports. The public repo has the schema (v4+) but no documentation on how the viewer uses it.
- **v7 export schema**: Not present in the public repo at all — only v0–v6 export parsers exist. The exact Zod schema for v7 is unknown.

## 3. Items

Every item has these **base fields**:

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | `string` (non-empty) | **yes** | UUID |
| `type` | see table below | **yes** | Discriminator |
| `position` | `{x: number, y: number}` | **yes** | Canvas coordinates in pixels |
| `size` | `{width: number, height: number}` | **yes** | Item dimensions in pixels |
| `dropShadow` | `boolean` | **yes** (v1+) | |
| `title` | `string` or `null` | no | Display title near the item |
| `groupId` | `string` or `null` | no | Links item to a group |
| `notes` | `string` or `null` | no | User annotations |
| `customThumbnail` | `string` or `null` | no | User-uploaded override thumbnail (URL or `file:/`) |
| `thumbnail` | varies (see §3.1) | no | Auto-generated thumbnail(s) |

### Valid `type` values

| `type` | Description | Extra required fields |
|---|---|---|
| `text` | HTML-formatted text frame | `text` (string), `backgroundColor` (hex or null) |
| `image` | Static image | `source` (string: URL or `file:/`) |
| `audio` | Audio clip | `source`, plus `startTime`/`stopTime` (number or null) |
| `video` | Video clip | `source`, plus `startTime`/`stopTime` (number or null) |
| `book` | E-book | `source` |
| `pdf` | PDF document | `source`, plus `defaultPage` (int or null) |
| `webpage` | Embedded webpage | `source`, plus `webpageType` (`"youtube"`/`"vimeo"`/`"iaWayback"`/`"iaAudio"`/`"iaVideo"` or null), `timestamp` (string or null) |
| `actionButton` | Clickable button | `actionType` (`"internalLink"` or `"externalLink"`), `text` (HTML string), `action` (string or null), `backgroundColor` (hex or null) |

### 3.1 Thumbnail field — two formats

The `thumbnail` field on items can be **null**, or one of two shapes:

**Simple format** (v6 spec, used by converters):
```json
{
  "source": "file:/items/uuid-thumbnail (name).jpg",
  "size": { "width": 400, "height": 300 }
}
```

**Renditions format** (v7 production, auto-generated by Tapestries):
```json
{
  "renditions": [
    {
      "source": "file:/items/uuid-auto-primary-thumbnail-webp-2000x174 (...).webp",
      "format": "webp",
      "size": { "width": 2000, "height": 174 },
      "isPrimary": true,
      "isAutoGenerated": true
    },
    {
      "source": "file:/items/uuid-auto-derived-thumbnail-webp-256x22 (...).webp",
      "format": "webp",
      "size": { "width": 256, "height": 22 },
      "isPrimary": false,
      "isAutoGenerated": true
    }
  ]
}
```

### ⚠️ Unknowns (thumbnails)

- The **renditions** format is not documented in the public repo at all — no Zod schema, no TypeScript type. It is only known from reverse-engineering a v7 export.
- The set of `format` values (observed: `"webp"`, `"jpeg"`).
- Which item types can have thumbnails: in the repo schemas, `text` and `actionButton` types do **not** have a `thumbnail` field, but v7 exports include thumbnail renditions on **all** item types including `text` and `actionButton`. This is a **v7 change** not reflected in the public source.
- Whether the simple format is still accepted on import (likely yes, since the export parsers chain through all versions).

## 4. Rels (Relations / Arrows)

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | `string` | **yes** | UUID |
| `from` | `RelEndpoint` | **yes** | |
| `to` | `RelEndpoint` | **yes** | |
| `color` | hex color string | **yes** | Line color |
| `weight` | `"light"`, `"medium"`, or `"heavy"` | **yes** | v4+ |

**RelEndpoint:**
| Field | Type | Required | Notes |
|---|---|---|---|
| `itemId` | `string` | **yes** | Must match an item `id` |
| `anchor` | `{x: number, y: number}` | **yes** | Values in [0,1]; (0,0)=top-left, (1,1)=bottom-right of the item's bounding box |
| `arrowhead` | `"none"` or `"arrow"` | **yes** | |

## 5. Groups

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | `string` | **yes** | UUID |
| `color` | hex color or `null` | no | Border/background color |
| `hasBorder` | `boolean` | **yes** | |
| `hasBackground` | `boolean` | **yes** | |

Items reference groups via their `groupId` field.

## 6. Hex Colors

The Tapestry app validates hex colors as: **starts with `#`, any length**. In practice, these forms appear:

| Form | Example | Where used |
|---|---|---|
| `#RRGGBB` | `#f8f9fa` | Common |
| `#RRGGBBAA` | `#ffffff00` | Transparent backgrounds in Tapestries-authored items |

## 7. Version History (from public repo)

| Version | Key changes |
|---|---|
| v0 | Original format: items, rels, background, theme. `title` was required (non-null). |
| v1 | Added `tapestryId`, `dropShadow` (required boolean), `skipSeconds` (audio/video/webpage). `createdAt`/`updatedAt` required at root. |
| v2 | Removed `tapestryId` from export. Added `description`, `parentId`, `startView`, `thumbnail` at root. `backgroundColor` on text items. `internallyHosted` flag. |
| v3 | Replaced `skipSeconds` with `startTime`/`stopTime`. Added `PlayableMediaItemSchema`. |
| v4 | Made `title` nullable. Added `groupId`, `notes`, `customThumbnail`. Added `groups` and `presentation` arrays at root. Added `weight` to rels. `defaultPage` on PDF. `webpageType`, `timestamp` on webpage. |
| v5 | Added `actionButton` item type. PDF `thumbnail`. |
| v6 | `actionButton.actionType` refined to `"internalLink"`/`"externalLink"`. |
| **v7** | **NOT DOCUMENTED.** Added `parentId` at root? Rich `renditions`-based thumbnails. Thumbnails on all item types including `text`/`actionButton`. |

## 8. Open Questions / Unknowns

1. **v7 export schema**: No source code exists in the public repo. The schema is only known from a single production export file.

2. **Renditions format**: The `{renditions: [{source, format, size, isPrimary, isAutoGenerated}]}` structure has no Zod schema or TypeScript type anywhere in the public repo. Unknowns:
   - Full set of `format` values (observed: `"webp"`, `"jpeg"`)
   - Whether `isPrimary`/`isAutoGenerated` are always present
   - Whether the simple `{source, size}` format is still valid on import for v7

3. **Thumbnails on text/actionButton items**: The v6 repo schemas do not include `thumbnail` on `text` or `actionButton` items. v7 exports include them on every item type. Unknown whether import accepts/generates them for all types.

4. **`parentId`**: Present in v7 export. Likely for tapestry forks. Semantics undocumented.

5. **`presentation` steps**: Schema exists (v4+ in the repo) but no behavioral documentation. The array establishes an ordered sequence (`prevStepId` linked list) pointing at items or groups.

6. **`internallyHosted` field**: Present in the DTO layer (v2) but stripped by the exporter. Unknown whether it should be included in exports or always removed. The exporter's `@ts-expect-error` on `delete item.tapestryId` suggests extra fields are routinely stripped.

7. **Tapestry-level thumbnail structure**: The root `thumbnail` is `string | null` (a URL). Does it ever use a renditions format at the root level? Unknown.

8. **Zod strictness on import**: The export parsers use `safeParse` which silently returns `null` on any validation failure. Extra fields are ignored, but missing required fields cause rejection. There is no friendly error reporting — this is why uploads fail silently.

---

## 9. How to Validate

The only practical approach today is to validate against what's observed in working files:

1. `root.json` must exist and parse as JSON
2. All required fields per §2 must be present with correct types
3. Every item `id` must be unique; every item must have a valid `type`
4. Every rel's `from.itemId` and `to.itemId` must reference existing item IDs
5. Every `file:/` reference must resolve to a file present in the zip
6. Hex colors must start with `#` (any length accepted)
7. `createdAt` and `updatedAt` are both required (a common omission)

The `validate-tapestry.py` script in this directory implements these checks and handles both thumbnail formats.
